Deployment

Use a single command to deploy your Next.js project.

next-yc deploy --project . --verbose

Default mode executes project analysis, build, artifact upload, and terraform apply.

Useful flags:

  • --dry-run or NYC_SKIP_BUILD=1 for non-mutating dry runs.
  • --skip-build to skip the application build phase.
  • --auto-approve to skip Terraform apply confirmation.
  • --app-name <name> to set the deployed application name.
  • --env <environment> to set the deployment environment.
  • --region <region> to target a specific Yandex Cloud region.
  • --assets-bucket <bucket> to specify the Object Storage bucket for static assets.

The build phase runs your project build script automatically when available.

Runtime config is read from next-yc-cfg.json (public/env/secrets/scope).

The CLI executes internal phases:

  1. analyze
  2. build
  3. upload
  4. terraform apply
  5. summary
